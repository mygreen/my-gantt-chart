# AGENTS

このファイルは、このリポジトリで作業する AI / コーディングエージェント向けの実務ガイドです。
実装方針の詳細は [SKILLS.md](SKILLS.md) を参照してください。こちらでは役割、優先順位、レビュー観点をまとめます。

## このプロジェクトの目的

このプロジェクトは、React + TypeScript + Spring Boot によるガントチャート Web アプリです。
単なるチャート表示ではなく、実務で使う編集性、保存性、比較性を重視しています。

特に重要なのは次の点です。

1. UI / UX を壊さずに機能追加すること
2. frontend と backend の責務を崩さないこと
3. 保存、版管理、休日、メンバー、プロジェクト管理を一貫した状態で扱うこと
4. 既存の画面挙動にそろえること

## 役割

### Frontend

担当:

* React / TypeScript / Zustand による UI 実装
* ガント、タスク一覧、ダイアログ、設定画面の編集
* SVG 出力、イナズマ線、表示切替のような描画系機能

原則:

* 状態は `stores/` に寄せる
* API 呼び出しは `api/` に寄せる
* レイアウト、日付計算、描画補助は `core/` に寄せる
* 文字化けを起こさない
* 画面と出力物の挙動をそろえる

### Backend

担当:

* Spring Boot の REST API
* DTO / Service / Repository / Entity の整理
* 保存、復元、プロジェクトコピー、版管理
* H2 / Flyway による初期化

原則:

* API 仕様を安易に壊さない
* DB の変更は migration とセットで扱う
* 既存の保存データと整合する実装を優先する
* 文字コードは UTF-8、BOM なしを前提にする

### Review

優先順位:

1. ビルドが通るか
2. 保存と再読み込みが壊れないか
3. SVG 出力と画面表示が一致しているか
4. 既存 UI の操作感が崩れていないか

## 作業ルール

### 実装前

* 既存の配置と naming に合わせる
* 同じ責務の処理がどこにあるかを確認する
* 画面機能は frontend だけで閉じるか、backend 変更が必要かを先に判断する

### 実装中

* 小さく変更する
* 文字化けした文字列を入れない
* 日本語ラベルは UTF-8 の通常文字で扱う
* 画面用データと SVG 出力用データのずれに注意する

### 実装後

* frontend は `npm run build` を通す
* backend は必要なら wrapper 経由でビルド確認する
* 仕様変更が利用者向けに見える場合は README 更新を検討する

## 特に注意するポイント

* プロジェクト単位の設定とシステム共通設定を混同しない
* プロジェクト休日とシステム祝日を混同しない
* バージョン復元は「過去版に戻す」ではなく「過去版の内容で新しい版を作る」挙動である
* SVG 出力は画面と同じ見え方を目指す
* 折りたたみ中の No. は全体ツリー順の番号を維持する

## 更新判断

次のときは docs 更新を検討します。

* 画面構成が増えた
* 保存仕様が変わった
* build / run 手順が変わった
* backend の技術構成が変わった
* SVG 出力や版管理のような重要機能が増えた
